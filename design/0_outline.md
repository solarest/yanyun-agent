# yanyun-agent 项目大纲

## 1. agent

### 1.1 agent 定义
- **agent.md** — Agent 核心配置与元信息
  - 说明：定义 Agent 的名称、版本、能力描述、输入输出规范
  - 注意：保持简洁，避免过度配置；与 soul.md 职责分离

- **role.md** — Agent 角色与行为边界
  - 说明：定义 Agent 扮演的角色、职责范围、权限边界
  - 注意：明确"能做什么"和"不能做什么"；与安全规范对齐

- **soul.md** — Agent 人格与风格定义
  - 说明：定义 Agent 的语言风格、性格特征、交互偏好
  - 注意：使用正向描述；避免模糊的性格定义

- **memory.md** — Agent 记忆使用说明
  - 说明：定义 Agent 如何使用记忆系统、记忆优先级、遗忘策略
  - 注意：与 memory 系统模块解耦，此处只定义行为规则

- **Agent 版本管理** — Agent 配置的版本控制
  - 说明：对 agent.md 等配置文件进行版本追踪
  - 注意：兼容性声明；变更日志；支持回滚

- **输入输出契约** — Agent 的接口约束
  - 说明：定义 Agent 的标准输入格式和输出格式
  - 注意：使用 Pydantic Schema 严格约束；向后兼容

### 1.2 prompt builder
- **分层构建** — 按层次组装 Prompt
  - 说明：系统提示词、角色提示词、任务提示词分层组装
  - 注意：各层职责清晰，避免耦合；支持热插拔

  - **前缀固定** — 系统级不变内容
    - 说明：安全约束、格式要求等固定前缀
    - 注意：精简长度，避免浪费 Token；定期评估有效性

  - **schema 定义** — 输出格式约束
    - 说明：定义 Agent 输出的 JSON Schema 或结构化格式
    - 注意：使用 Pydantic 定义，与领域层 DTO 对齐；严格验证输出格式

  - **Prompt 版本管理** — 提示词模板版本控制
    - 说明：对 prompt 模板进行版本管理，支持 A/B 测试
    - 注意：模板与代码分离存储；版本号语义化；变更影响评估

  - **Few-shot 管理** — 示例库维护
    - 说明：管理和维护 few-shot 示例，按场景动态选择
    - 注意：示例质量审核；示例去重；相关性排序

  - **动态组装策略** — 按需组装 Prompt
    - 说明：根据任务类型、上下文动态选择 prompt 组件
    - 注意：组装逻辑可配置；避免组合爆炸；性能可控

### 1.3 agent-loop
- **核心循环** — Agent 思考-行动-观察的主循环
  - 说明：基于 LangGraph 实现状态机驱动的执行循环
  - 注意：状态管理清晰；支持中断与恢复；错误处理完善

#### 1.3.1 context 管理
- **裁剪策略** — 控制上下文长度
  - 说明：根据 Token 限制裁剪过长的对话历史
  - 注意：保留关键信息；优先保留最近对话；避免截断中间关键内容

- **压缩策略** — 上下文摘要压缩
  - 说明：对历史对话进行摘要，减少 Token 消耗
  - 注意：摘要质量影响后续推理；定期评估压缩效果

- **注入策略** — 动态信息注入
  - 说明：将记忆、工具结果、外部信息按需注入上下文
  - 注意：注入位置固定；避免信息过载；控制注入频率

#### 1.3.2 ReAct 实现
- **loop 主流程** — Reason + Act 循环
  - 说明：思考 → 选择工具 → 观察结果 → 继续思考的循环
  - 注意：每个步骤有明确的输入输出；支持并行工具调用；异常时优雅降级

- **Checkpoint/Resume** — 状态检查点与恢复
  - 说明：LangGraph checkpoint 机制，支持暂停→恢复执行
  - 注意：检查点持久化策略；恢复时状态一致性；内存占用控制

- **Cancel 信号处理** — 取消与中断
  - 说明：响应用户取消请求，安全中断当前执行
  - 注意：中断后清理资源；部分结果保留；取消原因记录

- **Node 级可观测性** — 图节点级别监控
  - 说明：每个 LangGraph node 的执行时间、输入输出、异常追踪
  - 注意：与监控系统集成；性能瓶颈识别；异常节点告警

#### 1.3.3 Harness 工程
- **卡主策略** — 处理 Agent 卡住的情况
  - 说明：检测 Agent 长时间无进展时的干预机制
  - 注意：超时阈值合理设置；干预方式渐进式（提醒 → 引导 → 终止）

- **防死循环** — 避免无限循环
  - 说明：检测重复模式、循环调用并主动打断
  - 注意：循环检测窗口合理；记录循环原因用于优化

- **兜底策略** — 异常情况下的安全措施
  - 说明：所有策略失效时的最终安全输出
  - 注意：兜底输出友好；记录完整上下文用于排查

### 1.4 tools
- **工具注册与发现** — 工具的统一管理机制
  - 说明：工具注册表、能力描述、参数 Schema
  - 注意：遵循 DDD 架构，工具接口定义在领域层，实现在基础设施层

- **web search** — 网络搜索工具
  - 说明：调用外部搜索引擎获取实时信息
  - 注意：搜索结果需验证来源可靠性；控制搜索频率；处理搜索失败

- **function call** — 函数调用工具集
  - 说明：Agent 可调用的内置函数集合

  - **file** — 文件操作
    - 说明：读取、写入、搜索文件内容
    - 注意：限制可访问路径；敏感文件保护；操作前确认

  - **clarify** — 澄清提问
    - 说明：当需求不明确时向用户提问
    - 注意：问题简洁聚焦；避免连环追问；提供选项降低用户负担

  - **plan** — 任务规划
    - 说明：将复杂任务分解为可执行步骤
    - 注意：步骤粒度适中；支持动态调整；记录规划依据

- **mcp** — Model Context Protocol 集成
  - 说明：通过 MCP 协议接入外部工具和服务
  - 注意：MCP Server 连接管理；超时和重试；版本兼容

- **skills** — 技能系统
  - 说明：预定义的复杂任务执行流程
  - 注意：技能定义与实现分离；支持技能组合；技能间无状态依赖

- **sub-agent** — 子 Agent 委派
  - 说明：将特定任务委派给专门的子 Agent 执行
  - 注意：委派边界清晰；结果汇总机制；错误传递处理

- **工具执行超时** — 防范工具长时间阻塞
  - 说明：每个工具调用设置独立超时时间
  - 注意：超时后返回部分结果；不同类型工具不同超时策略

- **工具沙箱执行** — 隔离执行环境
  - 说明：高风险工具在沙箱环境中执行（如代码执行、文件操作）
  - 注意：网络隔离；文件系统限制；资源配额（CPU/内存）

- **工具调用限流** — 防止工具滥用
  - 说明：对高频工具调用实施频率限制
  - 注意：不同工具不同限流策略；限流后优雅降级通知用户

### 1.5 memory 系统
- **本地记忆系统（存储+召回）** — Agent 的持久化记忆
  - 说明：基于向量数据库的记忆存储和语义召回
  - 注意：存储格式统一；召回相关性排序；定期清理过期记忆

- **daily memory** — 每日记忆汇总
  - 说明：每天结束时自动汇总当天重要交互和决策
  - 注意：汇总质量评估；避免信息冗余；支持按需查询

- **dream memory** — 深度记忆整合
  - 说明：定期对记忆进行深度整理和关联建立
  - 注意：整理频率合理设置；避免过度整理丢失细节；保留原始记忆

- **记忆重要性评分** — 记忆优先级机制
  - 说明：根据交互频率、交互深度、用户显式标记等维度评分
  - 注意：评分算法透明可调；高优先级记忆不被裁剪；衰减曲线可配置

- **情景记忆与语义记忆** — 记忆类型区分
  - 说明：区分"什么时间发生了什么"（情景）和"通用知识"（语义）
  - 注意：两种记忆分别存储和召回；语义记忆支持知识图谱关联

- **记忆整合触发** — 整合时机与调度
  - 说明：定义 dream memory 和 daily memory 的触发条件
  - 注意：避免高峰时段整合；支持手动触发；整合进度可观测

### 1.6 multi-agent 系统
- **多 Agent 协作** — 多个 Agent 协同完成任务
  - 说明：Agent 间的通信、任务分配、结果整合机制
  - 注意：角色分工明确；避免重复工作；冲突解决策略

- **通信协议** — Agent 间消息传递
  - 说明：定义 Agent 间的消息格式和交互协议
  - 注意：消息异步处理；支持超时重试；消息追踪

- **编排策略** — 任务分发与协调
  - 说明：根据任务类型选择合适的 Agent 执行
  - 注意：负载均衡；失败重试；结果汇总

- **Supervisor 模式** — 监督者协调
  - 说明：Supervisor Agent 负责任务分解、Agent 调度、结果审核
  - 注意：Supervisor 单点问题；决策透明可追溯；支持人工升格

- **Agent 注册与发现** — 动态 Agent 管理
  - 说明：Agent 能力注册、健康检查、动态上下线
  - 注意：注册信息实时同步；心跳机制；失效检测

## 2. llm adaptor
- **大模型对接适配** — 统一的 LLM 调用接口
  - 说明：封装不同大模型厂商的 API，提供统一的调用接口
  - 注意：遵循 DDD 架构，接口在领域层，实现在基础设施层；支持多模型切换

- **模型路由** — 智能模型选择
  - 说明：根据任务类型、成本、性能要求选择合适模型
  - 注意：路由策略可配置；降级策略完善；成本监控

- **调用优化** — Token 和性能优化
  - 说明：批量调用、缓存、流式处理等优化手段
  - 注意：缓存一致性；超时控制；错误重试

- **计费与监控** — 用量统计和成本控制
  - 说明：记录每次调用的 Token 消耗和成本
  - 注意：实时监控告警；定期生成用量报告；异常调用检测

- **结构化输出** — 强制 JSON 输出
  - 说明：利用模型的结构化输出能力，确保响应格式符合 schema
  - 注意：schema 与领域层 DTO 对齐；解析失败重试；校验异常处理

- **模型降级链** — 模型故障自动切换
  - 说明：主模型不可用时自动降级到备用模型
  - 注意：降级顺序可配置；降级事件告警；自动恢复检测

- **Prompt 缓存** — 减少重复 Token 消耗
  - 说明：利用模型 Prompt Caching 能力缓存系统提示词
  - 注意：缓存失效策略；缓存命中率监控；多厂商适配

## 3. 通信协议
- **SSE 实现** — Server-Sent Events 流式通信
  - 说明：后端通过 SSE 向客户端推送流式事件
  - 注意：连接保活机制；断线重连；消息顺序保证

- **Schema** — 消息格式定义

  - **事件定义** — 系统事件类型
    - 说明：定义所有可能推送的事件类型（如思考、工具调用、结果等）
    - 注意：事件类型枚举完整；向后兼容；版本控制

  - **定义与解析** — 消息结构和解析逻辑
    - 说明：使用 Pydantic 定义消息结构，前后端共享 Schema
    - 注意：严格类型检查；解析失败处理；增量兼容

- **流式背压** — 客户端处理能力适配
  - 说明：客户端处理慢时，服务端控制推送速率
  - 注意：背压信号传递；缓冲区管理；超时丢弃策略

- **重连状态恢复** — 断线后上下文恢复
  - 说明：客户端重连后恢复丢失的流式消息
  - 注意：事件 ID 序号；未确认消息重发；幂等处理

## 4. 配置管理
- **环境变量** — 敏感配置管理
  - 说明：API Key、数据库连接等通过环境变量注入
  - 注意：绝不硬编码；提供 .env.example；敏感信息不入日志

- **配置加载** — 统一配置中心
  - 说明：使用 Pydantic Settings 加载和验证配置
  - 注意：启动时验证配置完整性；提供默认值；环境隔离

- **热更新** — 无需重启的配置变更
  - 说明：运行时动态更新非安全类配置（如限流阈值、日志级别）
  - 注意：变更通知机制；回滚支持；变更审计

- **功能开关** — Feature Flags
  - 说明：通过配置控制功能启用/禁用，支持灰度发布
  - 注意：开关状态缓存；降级开关；清理废弃开关

## 5. 数据持久化
- **关系型数据库** — 结构化数据存储
  - 说明：使用 SQLAlchemy ORM 存储用户、会话、任务等数据
  - 注意：遵循 DDD 架构，Repository 接口在领域层；迁移使用 Alembic

- **向量数据库** — 记忆和语义搜索
  - 说明：存储记忆向量，支持语义相似度召回
  - 注意：embedding 模型选择；索引优化；定期清理

## 6. 错误处理
- **异常层次** — 分层异常定义
  - 说明：领域层定义业务异常，表现层统一处理
  - 注意：异常不跨层传递；提供错误码；记录完整上下文

- **错误响应** — 统一错误格式
  - 说明：所有错误返回统一 JSON 格式
  - 注意：不暴露内部实现；提供用户友好提示

- **重试策略** — 指数退避重试
  - 说明：对临时性错误（网络超时、限流）进行指数退避重试
  - 注意：最大重试次数限制；退避因子可配置；幂等性保证

## 7. 监控与日志
- **结构化日志** — 可查询的日志格式
  - 说明：使用 JSON 格式，包含 trace_id、user_id 等上下文
  - 注意：不记录敏感信息；分级输出；支持日志聚合

- **指标监控** — 运行时指标收集
  - 说明：Token 消耗、响应时间、错误率等关键指标
  - 注意：暴露 Prometheus 指标；设置告警阈值

- **链路追踪** — 请求链路追踪
  - 说明：使用 trace_id 追踪完整请求链路
  - 注意：跨服务传递 trace_id；记录关键节点

- **LangGraph 追踪** — 图节点级别可观测
  - 说明：记录每个 graph node 的执行时间、输入输出、异常
  - 注意：与 LangSmith/LangFuse 集成；性能瓶颈识别；异常节点告警

- **Agent 专属指标** — 业务级监控
  - 说明：任务完成率、工具调用成功率、重试次数等 Agent 指标
  - 注意：指标维度合理；避免指标爆炸；与业务目标对齐

- **成本分析仪表盘** — Token 消耗可视化
  - 说明：按模型、用户、任务类型维度展示成本分布
  - 注意：实时更新；趋势预测；超预算告警

## 8. 会话管理
- **会话生命周期** — 创建到终止
  - 说明：会话创建、活跃、暂停、恢复、终止的完整生命周期
  - 注意：状态机管理；空闲超时自动终止；终止后资源清理

- **会话持久化** — 状态保存与恢复
  - 说明：将 LangGraph 状态序列化存储，支持跨请求恢复
  - 注意：序列化格式统一；增量保存减少开销；版本兼容

- **多轮对话上下文** — 跨轮次状态管理
  - 说明：在多轮对话中保持任务上下文和中间结果
  - 注意：上下文大小控制；过期策略；隐私清理

- **会话事件溯源** — 完整事件记录
  - 说明：记录会话中所有事件（用户输入、Agent 思考、工具调用）
  - 注意：事件存储高效；支持回放；用于审计和分析

## 9. 安全防护
- **Prompt 注入防护** — 防止恶意指令
  - 说明：检测和过滤用户输入中的注入尝试
  - 注意：分层防御（输入过滤 + 系统提示词隔离 + 输出校验）

- **内容安全过滤** — 输入输出审核
  - 说明：对用户输入和 Agent 输出进行安全审核
  - 注意：敏感词过滤；有害内容识别；审核日志保留

- **工具访问控制** — 最小权限原则
  - 说明：不同 Agent/用户有不同的工具调用权限
  - 注意：权限模型清晰；动态授权；操作审计

- **API 安全** — 接口层防护
  - 说明：JWT 认证、CORS、速率限制、请求签名
  - 注意：Token 过期策略；HTTPS 强制；安全头配置

## 10. 人机协作 (HITL)
- **审批流程** — 高风险操作确认
  - 说明：对文件删除、外部 API 调用等高风险操作需人工确认
  - 注意：审批操作分类定级；超时自动拒绝；审批记录留存

- **澄清中断** — 需求不明确时提问
  - 说明：Agent 遇到歧义时主动暂停请求用户澄清
  - 注意：问题精准聚焦；提供选项降低认知负担；避免频繁中断

- **手动干预** — 用户主动介入
  - 说明：用户可随时暂停、修改、接管 Agent 执行
  - 注意：干预点清晰；干预后状态一致性；操作可撤销
